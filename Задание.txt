1. Есть две системы:
Одна система выгружает данные о посещаемости в виде директории с папками за каждый месяц с CSV файлы посещаемости определенной дисциплины за этот месяц.
Формат CSV-файла
Номер студента - группа - столбцы для каждого дня по дисциплине (12.02, 19.02)

Вторая система по успеваемости выгружает данные в формате XML.
Номер студента - группа - ID дисциплины - Название дисциплины - Баллы

2. Данные перегоняются, строится ETL-процесс c помощью Apache NiFi, AirByte, [Apache Arrow, Apache Spark] [для перегона в озеро данных или в хранилище данных] [ORC, Iceberg, Parquet] (снимаются показатели хранимого места на диске и время работы ETL-процесса) (перегоняется в структуру, которая оптимизирована для OLAP-запросов)

3. Выполнить OLAP-запросы и анализируются показатели (временные показатели: сколько выполняются различные запросы по сортировке, группировке, выборке, агрегации данных);

4. Сделать дашборды (витрины данных) на основе результата OLAP-запроса (успеваемость групп, посещаемость студентов)

5. Проанализировать полученные показатели в каждом используемом стеке технологий

Условия тестирования: датасеты одного размера для каждого стека технологий.

Стеки технологий:
1. Можно ли обойтись без 2 шага и выполнить OLAP-запросы. Анализируется показатели.
2. Данные перевести в ClickHouse, выполнить OLAP-запросы. Проанализировать показатели
3. Используется 2 шаг со всеми возможными технологиями. Насколько лучше показатели?
4. Сравниваются форматы ORC, Iceberg, Parquet
